# Marketplace Design

**Date:** 2026-04-01

## Overview

A GitHub repository that serves as a personal Claude Code skills/plugin marketplace. It acts as a browsable catalog — the single source of truth is a YAML file, and the README is auto-generated from it.

## Files

### `plugins.yaml`
The registry data. Each entry has:
- `name` (string): short, slug-style identifier (e.g. `web-access`)
- `repo` (string): GitHub repo path, e.g. `eze-is/web-access`
- `description` (string): one-line description of what the plugin does
- `tags` (list of strings): categories for filtering/browsing (e.g. `[web, search]`)

Example:
```yaml
- name: web-access
  repo: eze-is/web-access
  description: Web browsing and search capabilities for Claude Code
  tags: [web, search]
```

### `generate.py`
A Python script that reads `plugins.yaml` and writes `README.md`. It outputs:
1. A title and short intro paragraph
2. A markdown table with columns: Name (hyperlinked to `https://github.com/<repo>`), Description, Tags

Run with: `python generate.py`

### `README.md`
Auto-generated by `generate.py`. Not manually edited. This is what visitors see when they browse the repo on GitHub.

### `CLAUDE.md`
Instructions for agents working in this repo:
- What this repo is (a Claude Code skills catalog)
- How to add a plugin: add an entry to `plugins.yaml`, then run `python generate.py` to regenerate `README.md`
- Field definitions for each `plugins.yaml` field
- Note that `README.md` is auto-generated and should not be edited directly

## Data Flow

```
plugins.yaml --> generate.py --> README.md
```

## Constraints

- No CI/CD required — `generate.py` is run manually before committing
- No web frontend — GitHub renders `README.md` as the browsable UI
- No install tooling — this is a catalog only; installation is handled by separate tooling (e.g. `skill-vendor`)
